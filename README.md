# Farm Water Accounting

A SvelteKit application for creating QField survey projects and visualizing farm water requirements.

## Tools

### Form creator

The guided form creator:

- Imports plot boundaries from GeoPackage or GeoJSON.
- Lets users keep or discard imported columns.
- Requires an ID column and area column, with options to generate `P-001`-style IDs and area in acres.
- Supports OSM, Esri World Imagery, MBTiles, and GeoTIFF basemaps.
- Provides editable default fields for farmer, crop, sowing date, irrigation, lateral, FIC, and conjunctive water source data.
- Supports custom text, numeric, date, boolean, value-map, dependent value-relation, attachment, and multi-select fields.
- Exports an offline-ready QGZ for QGIS and QField.
- Can upload the generated project to QFieldCloud using an API token and project name.

The QGZ is generated by a local FastAPI service. GeoPackage layers are written with GeoPandas/OGR, while large offline basemaps are inserted into the archive in the browser. Bundled files use QGIS `attachment:` paths and Web Mercator (`EPSG:3857`) to align farm plots with MBTiles and XYZ basemaps.

### Visualization

The visualization tool:

- Imports GeoPackage, GeoJSON, zipped shapefiles, or a folder containing shapefile components.
- Maps configurable crop, sowing-date, and area columns.
- Parses common sowing-date formats, including `dd/mm/yy`.
- Displays farm plots over Esri satellite imagery.
- Calculates monthly and total irrigation requirements.
- Groups plots by an attribute and applies high-visibility map colors.
- Switches the dashboard between individual plot, group, and overall summaries.
- Exports processed data to CSV, GeoPackage, and PDF.

## Requirements

- Node.js and npm
- Python 3 with virtual environment support
- No Docker is required

Python dependencies are installed into `api/.venv` from `api/requirements.txt`.

## Development setup

```bash
npm install
npm run setup:api
npm run dev
```
- SvelteKit/Vite: `http://localhost:5173`
- FastAPI: `http://127.0.0.1:8010` (routes under `/fwa-api/…`)

Vite proxies `/fwa-api` to FastAPI. If port 5173 is occupied, use the alternate URL printed by Vite.

Useful commands:

```bash
npm run dev:ui    # SvelteKit only
npm run dev:api   # FastAPI only
npm run check     # Svelte and TypeScript checks
npm run build     # production frontend build
```

## QGZ export flow

1. The browser compacts plot GeoJSON (kept columns only, rounded coordinates), gzip-compresses it, and posts to `/fwa-api/generate-qgz`.
2. If the payload is still large (or a single POST is rejected with 413), it uploads in ≤256 KiB chunks via `/fwa-api/generate-qgz/init|chunk|complete`.
3. FastAPI writes `data.gpkg` with farm plots and lookup layers.
4. FastAPI builds the QGIS project XML and returns a base QGZ.
5. The browser adds selected MBTiles or GeoTIFF files without uploading those large files to the API.
6. The completed QGZ can be downloaded or pushed to QFieldCloud.

> **Note:** Paths use `/fwa-api/…`, not `/api/…`. On the shared WELL Labs host, Django already owns `/api/` (e.g. `api/watershed/`).

Internet basemaps require connectivity in QGIS/QField. MBTiles and GeoTIFF files are bundled for offline use.

## Water-budget calculation

Each plot is matched to a crop in `total_water_needed.csv`. Water is scheduled in relative crop months starting from the plot’s sowing date (month 1 = sowing month). Plot area scales monthly irrigation depth into volume.

```text
water per acre (m³) = (depth in mm / 1000) × 4046.8564224
plot water (m³)     = water per acre × plot area in acres
```

Displayed and exported values are cubic metres, rounded to two decimal places. Missing or invalid area values default to one acre.

Supported budget crops include Paddy, Cotton, Groundnut, Pulses, Millets, Sunflower, Sesame, and Chilli. Unmatched crops display a warning instead of a schedule.

## Production notes

CodeDeploy ships the static `build/` plus the `api/` package. After install it:

1. Installs Nginx config (`client_max_body_size 512m`, `/fwa-api/` → `127.0.0.1:8010`) to **`/etc/nginx/conf.d/welllabs.conf`**, plus http-level **`/etc/nginx/conf.d/00-upload-limits.conf`**.
2. Creates a shared venv at `/opt/welllabs/api-venv` and installs `api/requirements.txt`.
3. Installs the systemd unit `welllabs-api.service` (uvicorn on port **8010**).
4. Restarts `welllabs-api` and reloads Nginx; ValidateService checks `/fwa-api/health` directly and via Nginx.

Form export no longer depends on a raised nginx body limit alone: the browser gzip-compresses config and, when needed, uploads in ≤256 KiB chunks.

A pre-API Nginx snapshot is kept as `devops/nginx/welllabs.conf.legacy`. Keep QFieldCloud tokens out of source control.

Useful on-server checks after deploy:

```bash
systemctl status welllabs-api nginx
curl -s http://127.0.0.1:8010/fwa-api/health
curl -s http://127.0.0.1/fwa-api/health
sudo nginx -T 2>/dev/null | grep -A2 'location /fwa-api'
sudo tail -50 /var/log/welllabs-deploy.log
```
